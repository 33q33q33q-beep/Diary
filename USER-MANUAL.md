# LifeTime Diary（私の日記）かんたんガイド

このアプリは **インターネットのブラウザだけ** で日記を書けます。  
スマホでもパソコンでも同じように使えます。

**この説明の PDF 版**は、同じフォルダにある **USER-MANUAL.pdf** です。印刷したいときに使ってください。

---

## まず知っておくこと（3つだけ）

1. **日記のデータは、最初はそのスマホ／パソコンの中** に保存されます。  
2. **別の機種とも同じ日記を見たい** ときは、あとで説明する「同期」を一度だけ設定します。  
3. **パスワードのような大切な文字列**（トークンなど）は、**知らない人に見せない・送らない** でください。

---

## 1. アプリの開き方

1. 配ってもらった **アドレス（URL）** をブラウザに入力するか、リンクをタップします。  
2. 例の形（参考）: `https://○○.github.io/○○/diary.html`  
3. **URL のつづりまちがい**に注意してください。似た名前のサイトを開くと、うまく動きません。

**おすすめのブラウザ**

- iPhone・iPad → **Safari**  
- パソコン → **Chrome** または **Edge**

ホーム画面に追加して使ってもかまいません。

---

## 2. 日記を書く（いちばんかんたん）

1. **カレンダー**の、書きたい **日付** をタップします。  
2. 気分や本文、写真などを入力します。  
3. **閉じる** と、内容は保存されます。

同期をまだしていなくても、この端末の中では日記は残ります。

---

## 3. 「同期」とは？（3つの選び方）

「同期」とは、**インターネット経由で、別のスマホやパソコンと同じ日記をそろえる** ことです。

アプリの左（またはスマホではメニュー）にある **⚙️ 設定変更** を開き、**同期の方式** から選びます。

| 選び方 | こんな人向け | ひとことで |
|--------|----------------|------------|
| **GitHub Gist** | 文章が中心で、写真は少なめ | GitHub というサービスにメモのように保存する |
| **Google ドライブ** | 写真もたくさん使いたい | Google のマイドライブにフォルダができる |
| **ローカルのみ** | クラウドは使いたくない | 自分で ZIP ファイルを取っておく |

**どれも難しければ**、まずは **ローカルのみ** で日記だけ始めて、あとから変更しても大丈夫です。

---

## 4. GitHub Gist で同期するとき（手順を順番に）

「GitHub」のアカウントが必要です（無料で作れます）。

1. パソコンで GitHub にログインします。  
2. **設定（Settings）** から **Personal access tokens（個人用トークン）** を新しく作ります。  
3. 作るとき、**gist** にチェックを入れます。  
4. 表示された **長い英数字（トークン）** をコピーします。**二度と同じ画面では見られない**ので、メモしておいてもよいです（**人に見せない**）。  
5. 日記アプリで **設定変更** → **GitHub Gist** を選び、トークンを貼り付けます。  
6. **Gist ID** は、**はじめてのときは空欄**のままで大丈夫なことが多いです。  
7. **保存して同期** を押します。

### スマホにも同じ設定をしたいとき

1. パソコンで **「📱 設定リンクをコピー」** を押します。  
2. 出てきた **長いアドレスを、先頭から最後までまるごと** コピーします。  
3. スマホの Safari で、そのアドレスを **アドレス欄に貼って開きます**。  
4. これでスマホ側にも設定が入ります。

**注意:** 設定リンクの中にトークンが入っています。**LINE やメールで他人に送らない** でください。

---

## 5. Google ドライブで同期するとき（ざっくり）

Google のアカウントと、少しだけ **Google Cloud**（開発者向けの設定画面）の操作が必要です。**いちばん手順が多い**ので、ゆっくりやってください。

くわしい画面つきの説明は、同じフォルダの **GDRIVE-SETUP.md** にあります。

### ざっくり流れ

1. Google Cloud のサイトで、**Google ドライブ API** を「使える」にします。  
2. **OAuth クライアント ID** を作ります。種類は必ず **ウェブアプリケーション** にします。  
3. **承認済みの JavaScript 生成元** に、**日記を開いているサイトのアドレスの「https://～」の部分** を、そのまま追加します。（配布者が教えてくれることが多いです）  
4. 日記アプリの **設定変更** → **Googleドライブ** で、**クライアント ID** を貼ります。  
5. **ログインしてフォルダ作成** を押し、Google の画面で許可します。

### データはドライブのどこ？

**マイドライブ** の中に、フォルダ **LifeTimeDiary** ができます。

- **diary-data.json** … 日記のデータ  
- **data** フォルダ … 写真など  
- **USER-MANUAL.pdf** … この使い方（PDF）。**同期が成功したあと**、ここに自動で置かれます。見当たらないときは、一度パソコンやスマホで日記を開いて **🔄 同期** を押してから、しばらく待ってドライブを開き直してください。

### iPhone でうまくいかないことが多いポイント

- **クライアント ID だけ** を Safari のアドレス欄に貼らないでください（壊れたアドレスになることがあります）。  
- **Drive設定リンクをコピー** した **https から最後まで全部** をアドレス欄で開く方法が安全です。  
- または、ID を **日記アプリの中の入力欄** にだけ貼ります。

---

## 6. クラウドを使わないとき（ZIP）

1. **📦 ZIP書出し** … 今の日記をまとめてファイルで保存します。  
2. **📂 ZIP読込** … 保存したファイルを選んで戻します。

iPhone では **ファイル** アプリや iCloud に保存しておくと安心です。  
**ときどき ZIP書出し** しておくと、機種が壊れても安心です。

---

## 7. よく使うボタン

- **🔄 同期** … いまクラウドとそろえたいとき。まだ設定していないときは、メッセージだけ出ます。  
- **⚙️ 設定変更** … 同期のやり方やパスワードのような設定。  
- **祝日・大切な日** … カレンダー上のボタンで、表示のオン・オフができます（最初はオン）。

書いたあと、同期の設定をしていれば、**数秒〜少し待つと自動でクラウドに送られる** ことが多いです。

---

## 8. うまくいかないとき

- **電波や Wi-Fi** がつながっているか確認する。  
- **設定変更** で、選んでいる同期の種類が思っているものと同じか見る。  
- Google のときは、**クライアント ID** をもう一度 Cloud の画面からコピーし直す。  
- **別の端末で内容が違う** → 両方とも **同じ Google アカウント／同じ Gist** になっているか、**同期ボタン** を押してみる。

---

## 9. 大切なお願い（セキュリティ）

- トークンや設定用の長い URL は **自分だけの秘密** にしてください。  
- このアプリは **個人で楽しむため** のものと考えてください。大切な思い出は、**ZIP やドライブのコピー** でも残しておくと安心です。
