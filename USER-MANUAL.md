# 私の日記（Webアプリ）— 渡し用マニュアル

**PDF版:** 同じフォルダの **[USER-MANUAL.pdf](./USER-MANUAL.pdf)** を印刷・配布に使えます。  
（中身を直したあと再生成する場合は `python3 scripts/build_user_manual_pdf.py` …要: `pip install reportlab`）

ブラウザだけで動く日記アプリです。データは **端末のブラウザ内** に保存され、オプションで **GitHub Gist** または **Google ドライブ** と同期できます。

---

## 1. 開き方

配布者が **GitHub Pages** などで公開している場合は、その **HTTPS の URL** をブックマークして使います。

- 例（形の参考）: `https://（ユーザー名）.github.io/（リポジトリ名）/diary.html`
- **ホスト名の打ち間違い**に注意（似た名前のサイトを開くと同期やログインが失敗します）。

**推奨ブラウザ**: Safari（iPhone）／Chrome や Edge（PC）。  
**ホーム画面に追加**しても利用できます（公開側の設定による）。

---

## 2. 初めて使うとき（流れ）

1. 上記 URL でアプリを開く。  
2. 左の **⚙️ 設定変更**（狭い画面ではカレンダー下のメニュー）を開く。  
3. **同期の方式** から次のいずれかを選び、指示に従う。

| 方式 | 向いている人 | 概要 |
|------|----------------|------|
| **GitHub Gist** | テキスト中心・データ量が小さめ | GitHub のトークンと Gist で同期 |
| **Google ドライブ** | 写真・メディアも多い | フォルダ `LifeTimeDiary` に JSON と圧縮ファイル |
| **ローカルのみ** | クラウドを使わない | **ZIP書出し／ZIP読込** で自分でバックアップ |

※ 同期を使わない場合でも、日記の入力は可能です（その端末にだけ保存）。

---

## 3. GitHub Gist で同期する場合

1. [GitHub → Personal access tokens](https://github.com/settings/tokens/new) でトークンを作成  
   - **gist** にチェック  
   - 失くさないよう **有効期限なし** も検討  
2. アプリの **設定変更** で **GitHub Gist** を選び、トークンを貼る。  
3. **Gist ID** は初回は空欄でよい（自動作成されることが多い）。  
4. **保存して同期** を押す。  

**別の端末（例: iPhone）**  
- Mac などで **「📱 設定リンクをコピー」** を押し、出てきた **URL 全体** を iPhone の Safari の **アドレス欄** に貼って開く。  
- トークンが URL に含まれるため **他人に送らない** でください。

---

## 4. Google ドライブで同期する場合

**自分用の Google アカウントと Google Cloud の設定が必要**です（無料枠で始められることが多いです）。

手順の詳細は同梱の **[GDRIVE-SETUP.md](./GDRIVE-SETUP.md)** を参照してください。要点だけ:

1. Google Cloud で **Drive API 有効化**、**OAuth クライアント ID（ウェブアプリケーション）** を作成。  
2. **承認済みの JavaScript 生成元** に、**実際に日記を開く URL のオリジン**（例: `https://ユーザー名.github.io`）を **完全一致** で追加。  
3. アプリの **設定変更** → **Googleドライブ** → **クライアント ID** を貼る → **ログインしてフォルダ作成**。  

**データの保存場所（マイドライブ）**  
- フォルダ **`LifeTimeDiary`**  
  - `diary-data.json` … 日記などのデータ  
  - `data/` … 写真などの圧縮ファイル  
  - **`USER-MANUAL.pdf`** … 使い方マニュアル（同期のたびに自動で置かれます。Drive アプリから開けます）  

**iPhone でクライアント ID を渡すとき**  
- **Drive設定リンクをコピー** した **https から最後まで丸ごと** を Safari で開く。  
- **クライアント ID だけ**をアドレス欄に貼らない（誤動作の原因になります）。  
- ID は **ウェブアプリ用**（iOS 専用クライアントではない）を使います。

---

## 5. ローカルのみ＋ZIPバックアップ

クラウドを使わない場合:

- **📦 ZIP書出し** … データを ZIP 化してダウンロード。iPhone なら **ファイル** アプリ経由で iCloud 等へ保存可能。  
- **📂 ZIP読込** … 保存した ZIP を選んで復元。  

定期的な ZIP 書出しを推奨します。

---

## 6. 日常の使い方

- **日記** … カレンダーの日付をタップして入力。閉じると **自動でクラウドへ送信**（同期設定済みの場合、数秒遅れで送る仕組み）。  
- **🔄 同期** … 手動で送受信したいとき。未設定のときは設定画面は開かずメッセージのみ出ます。  
- **⚙️ 設定変更** … 同期方式・トークン・クライアント ID。  
- **祝日・大切な日** … 初期状態は表示オン（カレンダー上のトグルで変更可）。  

**PC の Chrome** ではカレンダー表示が大きめになる画面設定になっていることがあります（マウス操作想定）。

---

## 7. トラブル時のチェックリスト

| 症状 | 確認すること |
|------|----------------|
| 同期できない | **設定変更** で方式・トークン／ログイン状態。通信（Wi-Fi） |
| Google でログインできない | **JavaScript 生成元** が **今開いている URL** と一致しているか。**クライアント ID はウェブ用**か |
| iPhone で ID が不正と言われる | メモから貼ると不可視文字が入ることがある。**欄を空にして貼り直し**、一度欄の外をタップ。または **Drive設定リンク** で開く |
| 404 になる | アドレス欄に **ID だけ**貼っていないか。**リンク全体**か **アプリ内の入力欄** を使う |
| 別端末でデータが違う | 両方で **同じ Gist／同じ Google アカウント** か。片方 **同期** を実行 |

---

## 8. セキュリティ・プライバシー（重要）

- **GitHub トークン**・**OAuth クライアント ID**・**設定用 URL** は **第三者に共有しない** でください。  
- 端末を **家族以外と共有**する場合、ブラウザに残ったデータに注意してください。  
- 本アプリは **個人利用・自己責任** を前提とします。配布者・作者はデータ消失等について **責任を負いかねます**。重要なデータは **ZIP やドライブ側** でもバックアップしてください。

## 10. 自分でホストする人へ（配布者向けメモ）

- リポジトリを Fork し、**GitHub Pages** で `diary.html` が開くようにする。  
- **Google ドライブ**を使わせる場合、利用者各自が **自分の Cloud プロジェクト**でクライアント ID を作るか、**同じクライアント ID** を共有するか決める（共有する場合は **承認済み生成元** に **実際の公開 URL** を必ず追加）。  
- 利用者向けには **このマニュアル（USER-MANUAL.md）と正しいアプリの URL** を渡すとスムーズです。
